## 1. Contexte

La CBAO souhaite mettre en place une dataplateforme moderne et scalable sur infrastructure Kubernetes on-premise pour supporter l'ensemble du cycle de vie des données : ingestion, transformation, stockage, analyse et gouvernance.

### Objectifs

- Architecture élastique et scalable basée sur Kubernetes
- Haute disponibilité et résilience
- Sécurité centralisée (Keycloak, OPA)
- Observabilité complète (Prometheus, Grafana)
- Gouvernance des données (Data Catalog)
- Environnement collaboratif (Zeppelin)
- Autonomie de l'équipe Data Engineer

---

## 2. Architecture

### Composants principaux

- **Orchestration** : Apache Airflow
- **Intégration** : Apache Nifi
- **Transformation** : DBT + Apache Spark
- **Stockage** : MINIO + Apache Iceberg
- **Moteur SQL** : Trino
- **Métadonnées** : PostgreSQL
- **Datalab** : Apache Zeppelin
- **Gouvernance** : OpenMetadata
- **Sécurité** : Keycloak + Open Policy Agent (OPA)
- **Monitoring** : Prometheus + Grafana + Loki
- **Backup** : Velero
